# Product UX Rules
The landing page is the visual and UX source of truth. All other pages must extend its branding, terminology, spacing, card style, dark theme, violet accent, and developer-tool tone.
This product is a GitHub repository review tool.

The user-facing promise is:
"Paste a GitHub repo. Choose what you want checked. Get a prioritized engineering report with file-level findings, evidence, and suggested fixes."

Always prefer user-facing terminology:
- Review, not Session
- Report, not Consensus
- Finding, not Artifact
- AI reviewer, not Agent
- Analyze repo, not Start round
- Activity log, not Replay

Do not expose internal workflow concepts unless inside advanced/debug views.

Primary UX path:
paste repo → choose review type → analyze repo → show progress → show report → copy/export fix plan.
